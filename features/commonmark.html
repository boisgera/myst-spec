CommonMark¶

This page provides an overview of the types of block and inline markup features supported by CommonMark and MyST, with pointers to additional content of interest. For full details on all the nuance of these features, please look at the CommonMark Spec documentation.

MyST (Markedly Structured Text) was designed to make it easier to create publishable computational documents written with Markdown notation. It is a superset of CommonMark Markdown and draws heavy inspiration from RMarkdown syntax. In addition to CommonMark, MyST also implements and extends mdast, which is a standard abstract syntax tree for Markdown. mdast is part of the unifiedjs community and has many utilities for exporting and transforming your content.

Block Markup¶

Headings¶

  • type*: string (“heading”) - See Node

  • depth*: integer

  • children*: array (PhrasingContent) - See Parent

  • identifier: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • position: object (Position) - See Node

  • data: object - See Node

### Heading!

type: root
children:
  - type: heading
    depth: 3
    children:
      - type: text
        value: Heading!

See also

Reference headings by preceding headers with a (label)=. See References & Links!

Lists¶

  • type*: string (“list”) - See Node

  • ordered: boolean - Is item order important or not?

  • start: integer - Starting number of ordered list

  • spread: boolean - One or more children are separated with a blank line from others

  • children*: array (ListContent) - See Parent

  • position: object (Position) - See Node

  • data: object - See Node

1. quotes
2. breaks
3. links

type: root
children:
  - type: list
    ordered: true
    start: 1
    spread: false
    children:
      - type: listItem
        spread: false
        children:
          - type: text
            value: quotes
      - type: listItem
        spread: false
        children:
          - type: text
            value: breaks
      - type: listItem
        spread: false
        children:
          - type: text
            value: links

  1. quotes

  2. breaks

  3. links

Code¶

Block of preformatted text

  • type*: string (“code”) - See Node

  • lang: string - language of the code

  • meta: string - custom information relating to the node

  • class: string - user-defined class for code block

  • showLineNumbers: boolean

  • startingLineNumber: integer

  • emphasizeLines: array (integer)

  • identifier: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • value*: string - See Literal

  • position: object (Position) - See Node

  • data: object - See Node

```python
print('this is python')
```

type: root
children:
  - type: code
    lang: python
    value: print('this is python')

print('this is python')

See also

Create code-blocks with additional highlighting using the code-block directive. See more here!

  • title: string - See Resource

  • position: object (Position) - See Node

type: root

    label: Key
